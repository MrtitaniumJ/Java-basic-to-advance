## Content Guidelines

### Language Style
- **Simple and Clear**: Explain like teaching a friend
- **No Jargon**: Use everyday language first, then technical terms
- **Examples First**: Show code, then explain concepts
- **Progressive Complexity**: Start simple, add complexity gradually

### Code Examples Requirements
- **Working Code**: Every example should compile and run
- **Comments**: Explain what each part does
- **Real-World Context**: Banking, shopping, student systems
- **Error Handling**: Show common mistakes and fixes

### Visual Elements
- **ASCII Diagrams**: For concepts like inheritance, memory layout
- **Code Flow**: Step-by-step execution visualization
- **Comparison Tables**: Feature comparisons, syntax differences
